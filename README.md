# DD Marcket (땅땅마켓) 🔨
> **개발 기간:** 2025.10.14 ~ 2025.12.16

실시간 양방향 통신 기술로 경매의 현장감을 구현하고, 국세청 API 기반의 사업자 검증 시스템을 통해 투명한 거래 환경을 제공하는 경매·중고거래·스토어 올인원 커머스 서비스입니다.

---

## 🛠 기술 스택 (Tech Stack)

| 분류 | 기술 기술 및 도구 |
| :--- | :--- |
| **Languages** | Java, TypeScript, Python |
| **Frontend** | React (Vite, TailwindCSS) |
| **Backend** | Spring Boot, WebSockets, MySQL, JPA |
| **AI** | RAG (LangChain + OpenAI) |
| **Infra** | Git, Docker, Nginx, AWS (EC2, RDS, S3) |

---

## 📌 주요 업무 및 구현 기능

### 💬 실시간 커뮤니케이션
* **WebSocket 채팅 구현**
  * 판매자와 구매자 간의 실시간 1:1 채팅 기능 개발
  * 거래 상태(예약/판매완료) 연동 기능 구현

### 🔒 보안 및 인증 시스템
* **회원 인증 시스템 구축**
  * Spring Security와 JWT를 활용한 로그인/회원가입 및 토큰 기반 인증 구현
* **다중 신원 검증 로직**
  * SMS/이메일 본인인증 시스템 구축
  * 국세청 API 연동을 통한 사업자 진위 여부 실시간 검증

### 💻 풀스택 기반 기능 구현
* **초기 환경 구축**
  * React 프로젝트 초기 구조 설계
  * 상품/게시판 관리를 위한 핵심 CRUD API 개발
* **관리자 시스템**
  * 유저 관리 및 신고 처리를 위한 백오피스 기능 구현

## 🚀 트러블슈팅 (Troubleshooting)

### 1. 비즈니스 모델 확장에 따른 DB 한계
* **[Issue]** 초기 경매 중심의 설계로 인해 스토어, 중고거래 등 다양한 비즈니스 모델 확장에 한계가 발생했습니다.
* **[Solve]** DB 스키마 재설계를 진행하여 다양한 거래 유형을 통합 관리하고 유연하게 대처할 수 있는 확장성을 확보했습니다.

### 2. 서비스 확장(중고거래/스토어)에 따른 실시간 양방향 채팅 시스템 구축
* **[Issue]** 1번의 스키마 개편을 거쳐 중고거래 기능이 새롭게 도입되면서 판매자와 구매자 간의 원활한 소통 창구가 요구되었습니다. 기존의 일방향적인 '댓글' 방식은 흥정이나 실시간 문의를 처리하기에 시차가 발생하여 유저 경험을 저해하는 한계가 있었습니다.
* **[Solve]** 기능 확장 목적에 맞춰 상시 연결이 유지되는 웹소켓(WebSocket) 기술을 전격 도입했습니다. 주기적으로 데이터를 요청하는 HTTP 폴링 방식 대비 서버 리소스 트래픽 부담을 줄이면서, 별도의 새로고침 없이 메시지가 즉각 주고받아지도록 구현했습니다.

### 3. 팀원 참여도 불균형 및 개발 싱크 불일치
* **[Issue]** 팀원들의 불규칙한 참여로 인해 개발 싱크가 맞지 않고, 프로젝트 완성도가 저하될 우려가 있었습니다.
* **[Solve]** 이슈 제기와 함께 **'1일 2회 스크랩'** 문화를 정착시켜 진척도를 강제로 동기화하고 협업 효율성을 대폭 제고했습니다.
